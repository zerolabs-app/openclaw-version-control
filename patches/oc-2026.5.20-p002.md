# OpenClaw Patch oc-2026.5.20-p002

## Name

Recurring Workspace Session Enforcement Patch

## Applies To

- Mission Control: 1.0.0
- OpenClaw: 2026.5.20

## Purpose

- Enforce persistent workspace-bound recurring task execution
- Prevent isolated recurring sessions from bypassing Agent1 artifact creation
- Ensure recurring jobs generate canonical workspace artifacts
- Align recurring execution with Mission Control workspace-first architecture

## Patch Type

Runtime Prompt Patch

## Patch Target

~/.openclaw/workspace/mission-control/prompts/recurring-task.md

## Runtime Behavior Changes

- Recurring jobs now explicitly run inside:
  ~/.openclaw/workspace/agents/agent1-workspace/

- Recurring jobs must:
  - create task artifacts
  - create planning artifacts
  - create log artifacts
  - create event artifacts
  - create result artifacts
  - create feedback artifacts

- Recurring jobs must use:
  - persistent named sessions
  - workspace-bound session targets

- Isolated ephemeral sessions are no longer recommended for Mission Control recurring jobs.

## Session Rules

Recommended default session target:

session:agent1-recurring

Job-specific named sessions remain allowed.

Example:

session:tiktok-viral-report

## Delivery Rules

Delivery remains best-effort behavior.

Delivery failure is NOT task failure.

Workspace artifacts remain the operational source of truth.

## Patch Strategy

- Deterministic full-file replacement
- Non-destructive runtime override patch
- Preserve existing workspace artifacts
- Preserve existing cron jobs

## Files Modified

- ~/.openclaw/workspace/mission-control/prompts/recurring-task.md
- ~/.openclaw/workspace/OPENCLAW_PATCHES.json

## Expected Result

Future recurring jobs should:
- generate persistent Agent1 workspace artifacts
- appear correctly in Mission Control
- preserve recurring task history
- avoid isolated-session artifact loss
